# Grocery-Store-Management-System-using-SQL

## Introduction

This project implements a *Grocery Store Management System* using *SQL*.
It manages data related to customers, products, suppliers, employees, and orders.

Built as a *relational database system*, it demonstrates table relationships, data handling, and business analysis using SQL queries.

## Features

* Create and manage multiple related tables
* Use foreign keys and relationships
* Analyze customer purchasing behavior
* Evaluate product performance and revenue
* Track sales trends (daily & monthly)
* Measure supplier and employee performance

## Technology Stack

* SQL (MySQL)
* Joins & Aggregations
* Window Functions (RANK)

## Database Tables

* Supplier
* Categories
* Employees
* Customers
* Products
* Orders
* Order_Details

## Use Case

Helps analyze *retail grocery data* to understand sales, customers, and business performance.
